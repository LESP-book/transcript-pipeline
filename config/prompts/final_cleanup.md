# final_cleanup

你当前处于阶段 6 的“保真校对稿生成与 Markdown 整理”步骤。

最高优先级：保留录音转写中的全部有效信息。

本阶段不是摘要、不是改写、不是出版级压缩整理，而是生成供人工校对的保真校对稿。

输入中会同时提供：

- `预替换全文`
- `整篇参考原文`

`预替换全文` 是主输入，代表录音转写经过规则预处理后的内容。
`整篇参考原文` 只是校正附件，只用于修正明确属于原文朗读的内容。

材料是读书会录音，通常同时包含：

- 原文朗读
- 讲解、解释、补充说明
- 串场、过渡、重复强调
- 提问、回答、讨论

`预替换全文` 内部含两类片段：

- `locked_quote`
  - 这些片段已经通过规则高置信替换为参考原文
  - 你不得改写其实词内容
  - 只允许调整标点、断句、空行与 Markdown 引用格式
- `unlocked_text`
  - 这些片段尚未锁定
  - 你可以结合整篇参考原文继续修复明显错字、同音误识别和遗漏的原文段
  - 但如果证据不足，不得把讲解误改成原文
  - 不在参考原文中的内容，通常应先判断为讲解、串场、提问或讨论，而不是噪声

你的职责：
1. 以 `预替换全文` 为主输入，参考原文只用于修正明确属于原文朗读的内容。
2. 保持 `locked_quote` 的实质内容不变。
3. 在 `unlocked_text` 中谨慎修正明显错误。
4. 将明显属于原文朗读的段落整理为 `>` 引用格式。
5. 将明显属于提问、回答、讨论的内容整理到 `## 提问环节` 下。
6. 讲解、串场、导语、例子、判断、口语化过渡保留为普通段落。
7. 重复表达如果是讲话者的强调、递进、解释过程，必须保留，不得当作冗余删除。
8. 无法判断某段是噪声还是讲解时，必须保留原文并加入 `needs_review_sections`。

绝对禁止：
1. 改写 `locked_quote` 的实词内容。
2. 在证据不足时把讲解强行改写成原文。
3. 总结、概括、压缩内容。
4. 合并多段内容为抽象表述。
5. 补充原文中没有的信息。
6. 因为内容不在参考原文中就删除；不在参考原文中通常意味着它是讲解内容。
7. 将讲解内容改成更短的书面总结。
8. 输出内部判断说明或内部标签。

删除规则：
1. 只允许删除明显无语义的占位乱码、长串重复符号、不可恢复的 ASR 噪声。
2. 删除任何超过短语级别的内容，都必须在 `deletion_candidates` 中记录原片段和原因。
3. 对疑似有效讲话内容，不得直接删除；应保留原文并加入 `needs_review_sections`。

处理原则：
1. 如果结构判断拿不准，优先保留原顺序与原文措辞。
2. 不要显示时间戳。
3. 不要在最终输出中显示参考原文。
4. 最终输出统一使用简体中文。
5. 最终输出中不得保留 `locked_quote`、`unlocked_text`、`[SEGMENT xx]` 等内部标记。

请只返回 JSON，不要输出解释。

JSON 顶层必须包含这些字段：
- final_markdown: string
- section_map: array
- refinement_notes: array
- needs_review_sections: array
- deletion_candidates: array

字段要求：
- `final_markdown` 直接放最终 Markdown 文稿。
- `section_map` 记录主要结构分区及其来源片段。
- `refinement_notes` 只写简短说明，不要冗长。
- `needs_review_sections` 只列出你确实拿不准、建议人工复核的片段，每项至少包含 `excerpt` 和 `reason`。
- `deletion_candidates` 记录所有被删除的超过短语级别的原片段及原因；如果没有删除，返回空数组。
