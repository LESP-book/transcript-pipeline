# conversation_cleanup

你当前处于阶段 6 的“对谈录屏保真转录整理”步骤。

最高优先级：把录音转写整理成**有标点、有话轮、有合理分段、便于人工校对**的对谈稿，同时保留全部有效讲话信息。

本阶段不是摘要、不是采访稿再创作、不是出版级压缩整理，而是生成供人工校对的保真对谈整理稿。

输入只有录音转写全文，没有参考书目、参考原文或外部资料。

材料通常包含：

- 主持、访谈、讨论或自由对话
- 提问、追问、回答、插话、打断、补充说明
- 口语化过渡、重复强调、停顿后补充
- 开场、结尾、现场说明

## 必须完成的整理目标

1. **必须补全标点**：最终稿不能是无标点或少标点的连续长文本。
2. **必须合理分段**：每个自然话轮、问题、追问、插话、回答、补充说明都应尽量独立成段。
3. **必须显示对话结构**：能判断出问答、追问、插话时，必须在最终 Markdown 中体现出来。
4. **必须保留口语信息**：包括犹豫、转折、重复强调、临时补充、现场互动和解释过程。
5. **必须保留原意**：只做标点、断句、分段、轻度错字修正和结构整理，不得总结压缩。

## 标点规则

1. 根据语义补充逗号、句号、问号、感叹号、冒号、引号、顿号、分号等中文标点。
2. 疑问句、追问、反问必须使用问号。
3. 话题转折、插入说明、补充解释应使用合适标点断开，不能连成一整句。
4. 长句必须按语义拆成多个短句；不要输出一整段没有明显停顿的流水文本。
5. 如果 ASR 原文没有标点，也必须根据语义补标点。

## 分段与话轮规则

1. 一个自然话轮单独成段。
2. 提问单独成段，回答单独成段。
3. 追问必须单独成段，不得合并进上一段回答。
4. 插话、打断、临时补充必须单独成段，必要时标为“插话”或“补充”。
5. 同一说话人的长回答如果包含多个层次，也要按语义拆成多个短段。
6. 话题切换、举例、总结当前小点、转入下一问题时必须换段。
7. 最终 Markdown 不得只有一个大段落；如果输入较长，应形成多段清晰结构。

## 说话人标注规则

1. 如果 ASR 文本中明确出现人名、称呼或身份，例如“主持人说”“老师”“张老师”“某某问”，可以保留并整理为对应说话人标签。
2. 如果没有明确身份，但能从问答轮次判断不同说话人，可以使用中性标签：
   - `**说话人 A：**`
   - `**说话人 B：**`
   - `**说话人 C：**`
3. 不得凭空编造“主持人”“嘉宾”“老师”“学生”等身份。
4. 如果说话人只能根据话轮推断，必须在 `refinement_notes` 中写明：`说话人标签仅根据话轮推断，需人工复核。`
5. 如果无法判断是否换人，优先保留原文并自然分段；不要强行增加过多说话人。

## 推荐 Markdown 结构

优先使用下面这种结构组织 `final_markdown`：

```markdown
# 标题

**说话人 A：** 这里是一段已经补好标点的发言。

**说话人 B：** 这里是回答或接话。

**说话人 A（追问）：** 这里是追问，必须单独成段？

**说话人 B：** 这里继续回答。

**插话：** 这里是短暂打断或旁白补充。
```

如果原文中已有明确姓名或身份，可以替换“说话人 A/B”；如果没有明确依据，只能使用中性标签。

## 你的职责

1. 以录音转写全文为唯一主输入。
2. 添加必要标点、断句和空行，使内容适合人工校对。
3. 修正明确错字、别字、同音误识别。
4. 保留有效的问答、追问、解释过程、插话和重复强调。
5. 按话轮和语义层次重新分段。
6. 在不编造身份的前提下，尽量显示说话人或话轮标签。
7. 无法判断某段是噪声还是有效讲话时，必须保留原文并加入 `needs_review_sections`。

## 绝对禁止

1. 总结、概括、压缩内容。
2. 把多句对谈改写成抽象结论。
3. 为了通顺而替换原有说法。
4. 编造录音中没有的信息、背景、书名、人物身份或观点。
5. 删除有效讲话内容。
6. 输出无标点、少标点或整段堆叠的流水文本。
7. 把追问、插话、补充说明吞并到上一段里。
8. 输出内部判断说明或内部标签。

## 删除规则

1. 只允许删除明显无语义的占位乱码、长串重复符号、不可恢复的 ASR 噪声。
2. 删除任何超过短语级别的内容，都必须在 `deletion_candidates` 中记录原片段和原因。
3. 对疑似有效讲话内容，不得直接删除；应保留原文并加入 `needs_review_sections`。

## 质量自检

输出前必须自检：

1. `final_markdown` 是否已经补充了中文标点。
2. `final_markdown` 是否按话轮和语义分段。
3. 问题、追问、回答、插话是否尽量独立显示。
4. 说话人标签是否没有编造身份。
5. 是否没有摘要压缩或删除有效讲话。

如果自检发现标点或分段不足，必须先修正 `final_markdown`，再返回 JSON。

## 处理原则

1. 如果结构判断拿不准，优先保留原顺序与原文措辞。
2. 不要显示时间戳。
3. 最终输出统一使用简体中文。
4. 最终输出中不得保留 `[SEGMENT xx]` 等内部标记。

请只返回 JSON，不要输出解释。

JSON 顶层必须包含这些字段：
- final_markdown: string
- section_map: array
- refinement_notes: array
- needs_review_sections: array
- deletion_candidates: array

字段要求：
- `final_markdown` 直接放最终 Markdown 文稿，必须有标点、话轮和合理分段。
- `section_map` 记录主要结构分区及其来源片段。
- `refinement_notes` 只写简短说明；如果使用了推断说话人标签，必须记录需人工复核。
- `needs_review_sections` 只列出你确实拿不准、建议人工复核的片段，每项至少包含 `excerpt` 和 `reason`。
- `deletion_candidates` 记录所有被删除的超过短语级别的原片段及原因；如果没有删除，返回空数组。
