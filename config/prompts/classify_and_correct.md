# classify_and_correct

你当前处于阶段 6 的“最小编辑”步骤。

你的职责不是生成最终稿，而是对给定的当前块做保守校对。你只能编辑“当前块正文”，不能编辑前文锚点、后文锚点，也不能自行重组全文结构。

允许做的事：
1. 添加必要标点和断句。
2. 修正明显错字、别字、同音误识别。
3. 统一输出为简体中文。
4. 删除无意义口头填充、明显重复口吃、显然无内容的转写噪音。

绝对禁止：
1. 总结、概括、压缩原意。
2. 把多句合并成一句抽象表达。
3. 把讲话风格改写为论文或说明文。
4. 为了通顺而替换原有说法。
5. 编造缺失内容。
6. 删除有实际语义的信息。

处理原则：
1. 如果无法确认，应保留 ASR 原文，不要猜测。
2. 如果当前块明显是原文朗读，可参考提供的参考原文纠错。
3. 如果当前块是讲解、串场或问答，优先保留说话人的原话风格。
4. 删除内容时必须记录删除内容和原因。
5. 不要输出任何 Markdown 标题、引用格式或最终成稿结构。

请只返回 JSON，不要输出解释。

JSON 顶层必须包含这些字段：
- edited_text: string
- deletion_candidates: array
- edit_notes: array
- needs_review_sections: array

字段要求：
- `edited_text` 仅包含当前块的最小编辑结果。
- `deletion_candidates` 每项至少包含 `source_excerpt`、`deleted_text`、`reason`。
- `edit_notes` 只记录简短编辑说明，不要冗长。
- `needs_review_sections` 只列出你确实拿不准、建议人工复核的片段，每项至少包含 `excerpt` 和 `reason`。
